# Adapter: Local Agent / GUI 自动化

> 技术栈适配规则。项目按需启用；不属于通用核心。
> 来源：本机 GUI 自动化（桌面应用 UI Automation）实践蒸馏，不含任何项目事实。

## 架构底线

1. GUI 自动化只能发生在目标应用所在的那台机器上，由本机 Local Agent 执行；中心服务不直接操作 GUI。
2. Local Agent 默认只监听 `127.0.0.1`，禁止监听 `0.0.0.0`；页面调用"本机 Agent"必须用浏览器所在机器的 127.0.0.1，不走远程 API 基址。
3. 三禁底线（除非用户明确批准）：不读目标应用私有数据库、不 DLL 注入、不逆向私有协议。优先 UI Automation、视觉识别、OCR。

## 执行安全门

4. 前台焦点守卫：每次键鼠操作前验证目标窗口在前台，失败必须停止并回写原因。
5. hidden / minimized 窗口禁止自动恢复后继续业务操作，必须提示人工处理。
6. 未经目标验证（联系人 / 窗口标题 / OCR 确认）禁止粘贴或发送；partial match、需人工复核状态必须阻断。
7. OCR / 截图失败不得伪造成功结果。
8. 检测类链路保持只读：不写输入框、不按发送。
9. 真实发送必须有完整 gate 组合：目标验证、内容过滤、限频、失败回写、幂等、人工接管、紧急停止（全局热键 + API 双通道）。

## 任务与诊断

10. 任务按显式 task_id 指定执行，禁止依赖"队列头部"顺序；发送与检测任务类型互斥并共享运行锁。
11. 诊断接口不得返回原始 UIA 控件对象；必须安全序列化（仅保留 name/class/type/rect），异常返回结构化错误而非 500。
12. 无源码的测试机验收不得以"运行源码命令"为前提；以打包产物 + 页面操作验收。
