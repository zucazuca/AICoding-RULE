# Adapter: Docker / Compose

> 技术栈适配规则。项目按需启用；不属于通用核心。
> 来源：多环境 Compose 编排实践蒸馏，不含任何项目事实。

## 文件职责互斥

1. 每个 Compose 文件必须在文件头注释声明职责与边界：哪个是唯一生产入口、哪个是独立开发编排、哪个是 override 且禁止单独运行。
2. 独立编排与 override 禁止混用：dev 独立编排不得与生产主文件组合；override 文件不得单独 up。
3. staging / 第二环境必须完全隔离：独立 project name、独立端口段、独立数据卷、独立数据库名。

## Compose 语法坑

4. list 型字段（ports / volumes / env_file）在 override 中默认是 append 合并；完全替换必须用 `!override`，不要用 `!reset`（只清空不重设）。
5. env 文件通过 `--env-file` 显式传入，不依赖隐式加载；生产环境变量缺失时应快速失败，不允许静默回退开发默认值。

## 变更纪律

6. Compose / Dockerfile 属于高风险区域：修改前先完成风险分析（影响哪些环境、数据卷是否受影响、如何回滚）。
7. 涉及数据库容器的变更前先确认数据卷备份方式；init 脚本必须幂等，且显式指定用户/库名参数（不要依赖镜像默认值）。
8. 不把仅本机可用的路径 / 端口写进共享编排。
