# AI Project Reading Protocol

> 本文档是 AI Coding Agent 的项目阅读规范（AICoding-RULE 通用基线）。
>
> 本规范优先级高于编码规范、测试规范和输出规范。
>
> 在完成阅读和理解前，禁止直接生成代码。

------

# 1. 核心原则

AI 的首要职责不是写代码。

AI 的首要职责是：

```text
理解项目
理解业务
理解调用链
理解影响面
```

然后才是：

```text
设计方案
实现代码
执行测试
```

如果理解错误：

```text
后续全部错误
```

因此：

```text
Reading First
Coding Later
```

------

# 2. 阅读目标

阅读的目标不是"看过文件"。

阅读完成后必须能够回答：

```text
1. 这个项目是什么？

2. 当前需求属于哪个模块？

3. 当前数据从哪里来？

4. 当前数据写到哪里去？

5. 当前业务流程是什么？

6. 当前权限在哪里校验？

7. 当前会影响哪些模块？

8. 当前风险等级是什么？

9. 最小修改方案是什么？

10. 如何验证修改正确？
```

如果无法回答以上问题：

```text
继续阅读
禁止编码
```

------

# 3. 阅读流程

收到任务后必须按顺序执行：

```text
理解需求
↓
项目级阅读
↓
模块级阅读
↓
调用链阅读
↓
业务语义阅读
↓
影响面分析
↓
输出阅读结果
↓
进入方案设计
↓
进入实现
```

禁止跳过阅读阶段直接编码。

------

# 4. 项目级阅读

适用于：

```text
首次进入项目
陌生项目
大型重构
跨模块需求
```

优先阅读：

```text
项目入口文件（CLAUDE.md / AGENTS.md）

docs/ai/05_PROJECT_CONTEXT.md

README.md

requirements.txt
package.json
pyproject.toml
```

如果涉及以下内容：

```text
部署
环境变量
Docker
数据库
权限
认证
```

额外阅读：

```text
docker-compose 文件

项目环境变量模板（.env*.example 等）

migration

认证模块

权限模块
```

输出：

```text
# 项目理解

项目类型：

核心业务：

主要用户：

核心模块：

技术栈：

部署方式：

高风险区域：
```

禁止无目的遍历整个项目。

禁止无目的阅读整个 docs 目录。

------

# 5. 模块级阅读

定位当前需求所属模块。

例如：

```text
用户管理

权限管理

订单系统

知识库

Agent

RAG

支付

文件管理

通知系统

评测系统
```

必须确认：

```text
当前模块负责什么？

输入是什么？

输出是什么？

依赖哪些模块？

被哪些模块调用？

是否涉及：

权限
数据库
配置
外部服务
缓存
消息队列
```

输出：

```text
# 模块理解

模块职责：

输入：

输出：

依赖：

被调用方：
```

------

# 6. 调用链阅读

必须找出真实调用链。

后端常见调用链：

```text
API
↓
Service
↓
Repository / CRUD
↓
Database
```

前端常见调用链：

```text
View
↓
Component
↓
Store
↓
API
↓
Backend
```

AI 应用常见调用链：

```text
User Input
↓
Agent / Router
↓
Retriever / Tool
↓
Prompt
↓
LLM
↓
Parser
↓
Response
```

必须确认：

```text
入口在哪里？

核心逻辑在哪里？

数据读取在哪里？

数据写入在哪里？

返回结果在哪里？
```

输出：

```text
# 调用链

用户操作：

入口文件：

核心逻辑：

数据读取：

数据写入：

返回结果：
```

调用链不清晰：

```text
禁止修改
继续阅读
```

------

# 7. 文件阅读规则

修改前至少阅读：

```text
1. 当前入口文件

2. 核心逻辑文件

3. 数据模型或类型定义

4. 同目录相邻实现

5. 测试文件

6. 配置文件（如果任务涉及配置）
```

禁止：

```text
只读一个文件就修改

只看函数名猜测逻辑

只根据目录结构推断业务
```

------

# 8. 相邻实现优先原则

新增功能前：

必须寻找同目录、同模块、同类型实现。

优先学习：

```text
命名风格

代码组织方式

异常处理方式

日志方式

权限校验方式

返回结构

测试风格
```

原则：

```text
一致性优先于优雅性
```

禁止重新发明一套风格。

------

# 9. 业务语义阅读

必须区分：

```text
技术对象

业务对象
```

例如：

```text
current_user

creator

owner

admin

email
```

不一定代表：

```text
业务负责人

任务归属人

通知接收人
```

必须确认：

```text
操作人是谁？

业务负责人是谁？

数据归属对象是谁？

权限校验对象是谁？

通知接收对象是谁？

是否存在代操作场景？
```

禁止：

```text
使用最容易获取的数据

替代业务上正确的数据
```

------

# 10. 高风险区域阅读

以下属于高风险（通用基线，项目可在项目上下文中追加）：

```text
Docker

Docker Compose

Nginx

环境变量

数据库 Migration

认证

RBAC

支付

文件存储

缓存

异步任务

部署脚本

CI/CD

LLM Prompt 与真实模型调用
```

涉及高风险区域：

必须输出：

```text
# 高风险分析

当前实现：

修改原因：

影响范围：

风险等级：

验证方案：

回滚方案：
```

未完成风险分析：

```text
禁止修改
```

------

# 11. 数据库阅读规则

涉及数据库时：

必须阅读：

```text
Model

Schema

Repository / CRUD

Service

Migration

测试
```

必须确认：

```text
表结构

唯一索引

外键关系

历史数据

软删除逻辑

迁移方案

回滚方案
```

禁止：

```text
不了解表结构就修改数据库

不了解迁移历史就新增 Migration
```

------

# 12. 权限阅读规则

涉及权限时：

必须确认：

```text
有哪些角色？

权限在哪里校验？

前端是否仅隐藏按钮？

后端是否二次校验？

是否存在管理员代操作？

是否存在越权风险？
```

权限逻辑不清晰：

```text
禁止修改
```

------

# 13. AI / LLM 阅读规则

涉及：

```text
Prompt

Agent

RAG

Tool Calling

LLM
```

必须阅读：

```text
Prompt 模板

Prompt 版本

输入构造

检索逻辑

工具调用

输出解析

JSON Schema

Fallback 逻辑

测试样例
```

必须确认：

```text
输入是什么？

Prompt 如何构造？

输出是否结构化？

解析失败怎么办？

是否存在 Prompt Injection 风险？

是否需要稳定性测试？
```

------

# 14. 阅读结果输出

进入编码前必须输出：

```text
# 阅读结果

## 项目 / 模块理解

## 当前调用链

## 已阅读文件

## 业务语义确认

## 影响范围分析

## 风险等级

## 最小修改方案

## 验证方案
```

必须基于实际文件输出。

禁止：

```text
我已经理解项目

应该问题不大

看起来只需要小改
```

这类无依据结论。

------

# 15. 编码准入条件

只有同时满足以下条件：

```text
已完成阅读

已完成调用链分析

已完成业务语义确认

已完成影响面分析

已完成方案设计

已明确验证方案
```

才允许进入编码阶段。

否则：

```text
继续阅读
```

------

# 16. 最终原则

AI 必须先成为：

```text
项目读者
```

然后才能成为：

```text
项目开发者
```

在任何项目中：

```text
P0 阅读规范

P1 项目上下文

P2 执行规范

P3 测试规范

P4 输出规范
```

阅读规范拥有最高优先级。

------

# 17. 项目附加必读登记

项目可以在项目上下文（05_PROJECT_CONTEXT.md）中登记"附加必读"条目，格式：

```text
修改 <某类代码 / 某模块> 前，必读 <某验收文档 / 某专题文档>。
```

登记后的附加必读与本规范同等强制：

```text
不了解对应验收 / 边界状态的任务，
禁止修改该模块相关代码。
```

本节只定义机制；具体条目属于项目事实，禁止写入通用基线。
