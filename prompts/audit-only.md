# Prompt: 只审计不修改（audit-only）

> 用途：快速健康检查。严格只读，适合在不确定项目状态时先跑一轮。

------

## 任务

对项目 `{{PROJECT_PATH}}` 的 AI 规则与上下文做只读健康检查。

## 硬边界

- 禁止修改、移动、删除任何文件。
- 禁止 git 写操作。
- 输出只有报告。

## 检查清单

1. 规则文件完整性与受控区块版本。
2. CLAUDE.md 与 AGENTS.md 是否漂移（列差异）。
3. 05_PROJECT_CONTEXT.md：体量、是否流水账化、抽查 3~5 条结论与代码事实的一致性。
4. 归档纪律与 README 导航有效性。
5. 规则文件中的项目事实泄漏。
6. 文档腐化信号（追加式结论、状态并存）。

## 输出

- 每项：OK / 问题 + 证据（文件:行）。
- 风险分级：立即处理 / 下次维护处理 / 观察。
- 建议的后续任务（bootstrap / 基线重构 / 周期维护），不执行。
